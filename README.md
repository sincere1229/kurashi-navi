# 暮らしナビ v1.0(Phase 1 MVP)

引っ越しやることリスト自動生成サービス。設計・仕様書v1.0のPhase 1を実装。

## 実装済み機能
- 6問診断ウィザード(1問1画面・タップのみ)+引っ越し予定日入力
- タスク自動生成(条件フラグ判定・42タスク収録)
- マイチェックリスト:達成率バー/期日カウントダウン/カテゴリフィルタ/未完了絞り込み
- タスク詳細モーダル:窓口・持ち物チェック・ぴよ子アドバイス
- 完了時の「済」ハンコ演出+ぴよ子の褒めトースト
- localStorage保存(登録不要で使える)

## デプロイ手順(いつもの流れ)
1. GitHubで新リポジトリ作成(例:`sincere1229/kurashi-navi`)
2. このフォルダの中身をすべてアップロード(node_modulesは不要・自動生成されます)
3. Vercelで「New Project」→ リポジトリを選択 → そのままDeploy
4. 完了(環境変数は不要です)

## タスクの追加・修正
`lib/taskMaster.ts` を編集するだけ。1タスク=1オブジェクト。
- `condition`: 外側の配列=AND、内側=OR。`[]`は全員必須
  - 例:`[["B2","B3"],["C1"]]` →(市外 or 県外)かつ 車あり
- `deadline`: D-30 / D-14 / D-7 / D-1 / D0 / D+14 / D+30
- `advice`: ぴよ子のひとこと(🐣🐥は1メッセージ2個まで)

## v1.8 追加分(LINEメッセージの方針を明文化+リンク漏れ修正)

### 方針
LINEリマインドには**アフィリエイトリンクを直接入れない**。理由:
- 通知目的で登録してもらった信頼を壊すとブロック率が上がり、リマインド自体が届かなくなる
- LINE公式アカウントは「配信友だち数×配信回数」で課金されるため、リンクを増やしても通数コストへの影響はない(コストの問題ではなく信頼設計の問題)
- 収益化はチェックリスト・コラム画面側の`OfferBox`が担う。LINEは「戻ってきてもらうきっかけ」に専念する

### 修正内容
`lib/reminders.ts`の`reminderMessage()`が、実はチェックリストへのリンクを本文に含めていなかったため修正。今後すべてのリマインドメッセージ末尾に「▼ やることリストを見る」+ URLが自動で付くようになった。

### 参考:LINE公式アカウントのメッセージ通数(2026年時点)
- コミュニケーションプラン(無料):月200通まで
- ライトプラン(月5,000円):月5,000通まで
- スタンダードプラン(月15,000円):月30,000通まで+従量課金で追加可
- 友だちが増えてきたら無料プランの上限(200通/月)に達しやすいので、その際はプラン変更を検討

## v1.11 追加分(SEO本格実装:都市×手続きページ)

### 何を作ったか
「市区町村名+手続きキーワード」の検索で自サイトが表示されるよう、**外部検索リンクではなく自サイト内の実ページ**を大量生成する仕組みを実装した。

- `/guide/[都道府県]/[市区町村]/[トピック]` という動的ページを新設
- トピックは12種類(`lib/topics.ts`):転出入届・転居届・マイナンバー・国民健康保険・児童手当・免許住所変更・車庫証明・電気・ガス・水道・ゴミ分別・転校転園
- 全自治体(1,916)×12トピック=**約23,000ページ**が最終的に検索対象になる
- ビルド時は政令指定都市・県庁所在地・東京23区など**優先75市区×12トピック=896ページ**を事前生成(`lib/priorityCities.ts`)
- 残りの約1,840自治体は、**初回アクセス時に自動生成してキャッシュ**(ISR・60日ごとに再生成)。ビルド時間を抑えつつ最終的には全自治体をカバーする設計
- 各ページの中身は、既存の`taskMaster.ts`の実データ(窓口・持ち物・期限・アドバイス)を再利用しつつ、都市名・都道府県・公式サイトリンクなど**その自治体固有の実データ**を組み合わせているため、内容の薄い量産ページにならないよう配慮している
- `app/sitemap.ts`:23,000件超のURLを5,000件ずつ5ファイルに分割してサイトマップ出力(Next.jsのgenerateSitemaps機能)
- `app/robots.ts`:サイトマップの場所をクローラーに明示
- `/area`ページ:ガイドページでカバーしているトピックは外部Google検索ではなく自サイトの`/guide/...`への内部リンクに変更。カバー外(休日診療・学区等)のみ従来通り検索リンクで補完

### 運用時の注意
- Vercelのプラン次第ではISRの実行回数に制限がある場合があるため、アクセスが急増した際は確認する
- 今後トピックを追加する場合は`lib/topics.ts`に1項目追加するだけで、全自治体に自動展開される
- `lib/priorityCities.ts`の優先都市リストは、実際のアクセスログを見ながら追加・入れ替えるのがおすすめ

## v1.15 追加分(全タスク詳細画面にアフィリエイトを表示)

## v1.16 追加分【重要】/guideページの重大バグを修正+郡表記対応

### 発見した重大バグ(v1.11〜v1.15で発生していた)
`/guide/[pref]/[city]/[topic]`の全ページが、**ビルドは成功していたにもかかわらず、実際には全ページ404(エラー画面)を返していた。** 原因はNext.jsが日本語を含むURLパラメータ(都道府県名・市区町村名)をパーセントエンコード済みのまま渡し、`decodeURIComponent`せずに市区町村データと突き合わせていたため、一致判定が常に失敗していたこと。`findCity()`関数に`decodeURIComponent`処理を追加して修正。888ページ全てで正常に表示されることを確認済み。

**影響範囲:** v1.11以降でこの機能を紹介していたが、実際にはSEO効果が一切発生していなかったことになる。今回の修正により、初めて実際に機能する状態になった。

### 郡表記への対応
`lib/districtMap.ts`を新設。「都道府県+郡+市区町村」の正式表記(例:埼玉県比企郡嵐山町)をページタイトル・見出し・本文に反映できるようにした。現在は比企郡(嵐山町・小川町・川島町・吉見町・鳩山町・ときがわ町・東秩父村)のみ対応。全国の郡マッピングは別途データ整備が必要なため、今後追加していく方針。

### 「転居」キーワードの追加
ページの見出し・meta description・本文に「転居・引っ越し」という言い回しを追加し、「転居」で検索するユーザーとのマッチ率を上げた。

### 優先都市への追加
嵐山町を`lib/priorityCities.ts`に追加。デプロイ直後からインデックス対象になる。

## v1.17 追加分(転出元・転入先の任意登録+guideページへの内部リンク強化)

### 実装内容
チェックリスト画面に「転出元・転入先の街を登録する(任意)」カードを追加。診断フローには含めず、後からでも入力できる形にした(入力の手間を増やさないため)。

両方の市区町村を登録すると、以下3つの実ページへの直リンクが表示される:
- 転出元の「転出届・転入届まとめ」ガイド(`/guide/[転出元]/tenshutu-tennyu`)
- 転入先の「転出届・転入届まとめ」ガイド(`/guide/[転入先]/tenshutu-tennyu`)

### 設計意図(SEOとの関係)
「転出元×転入先」の組み合わせページは作成していない(市区町村同士の掛け合わせは検索需要が無く、天文学的なページ数になるため)。代わりに、アプリのメイン動線(チェックリスト画面)から既存の`/guide`単一都市ページへの内部リンクを増やす設計にした。これにより、実際のユーザー導線からのクロール・アクセスが増え、該当ページの評価向上が期待できる。

### データ構造
`lib/logic.ts`の`SavedState`型に`cities?: CitySelection`を追加(既存データとの後方互換性あり)。

## v1.20 追加分【重要な導線バグ修正】タスク詳細から/guideページへのリンクを追加

### 発見した問題
`/guide`ページ自体は正常に動いているが、**チェックリストのタスク詳細モーダルから`/guide`ページへの導線が一切無かった。** そのため「診断→チェックリスト→タスクを開く」という普通の利用フローでは、免許住所変更の警察署検索カードなど、`/guide`ページの情報に一生たどり着けなかった。

### 修正内容
`lib/topics.ts`に`topicByTaskId()`を追加(taskId→トピックの逆引き)。チェックリストのタスク詳細モーダルで、①該当タスクに対応するトピックがあり、②転入先の街が登録済み(`CitySelectorCard`で入力済み)の場合、「{転入先の街}の詳しいガイドを見る」というリンクが表示されるようにした。これにより、免許住所変更のタスクを開く→ガイドを見る→警察署検索カードに到達、という導線が普通の利用フローの中に組み込まれた。

### 今後の注意点
この導線は「転出元・転入先」を登録していないと出現しない。登録率が低いと導線として機能しないため、CitySelectorCardの入力率(現在は任意入力)は今後の課題として注視する必要がある。
